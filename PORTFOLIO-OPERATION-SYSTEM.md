# PORTFOLIO-OPERATION-SYSTEM.md

## 目的
将 `stock-learning-site` 从“研究内容系统”推进为“组合配置运行系统”。

说明：
- 本文件先作为组合层运行框架
- 不直接改写原始研究内容
- 后续可逐步把字段回填到 `frameworks/portfolio-template.md` 或组合实例文件中

---

## 一、组合层在系统中的位置
完整链路现在定义为：

**宏观变量 → 大类资产 → 知识判断 → 行业配置 → 个股观察 → 组合动作 → 周期复盘**

组合层负责解决的问题：
- 研究结论最终如何转化为配置动作
- 候选标的如何进入观察池 / 持仓池
- 为什么加仓 / 减仓 / 继续观察
- 复盘时如何验证判断是否有效

---

## 二、组合目标类型
### 1. 防守型组合
- 目标：降低波动，强调现金流与回撤控制
- 倾向资产：
  - 高股息权益
  - 国债 / 美债
  - 黄金
  - 现金
- 适合环境：
  - 风险偏好走弱
  - 增长预期下修
  - 利率下行或衰退交易

### 2. 平衡型组合
- 目标：兼顾收益与稳定性
- 倾向资产：
  - 核心权益
  - 债券
  - 黄金
  - 少量成长弹性
- 适合环境：
  - 宏观方向不明朗
  - 处于切换期

### 3. 进攻型组合
- 目标：追求更高收益弹性
- 倾向资产：
  - 成长行业
  - 周期修复行业
  - 高弹性个股
- 适合环境：
  - 风险偏好回升
  - 利率下行
  - 政策支持 / 盈利上修

## 四、候选持仓池机制
### 1. 候选池分层
#### A. 核心配置池
适合：
- 长期逻辑较稳
- 现金流质量高
- 可承担核心仓位

示例：
- `china-shenhua`
- `haitian`

#### B. 观察池
适合：
- 逻辑成立但时机未必成熟
- 需要继续验证行业或估值条件

示例：
- `cr-mixc`
- 半导体相关标的

#### C. 高弹性池
适合：
- 周期 / 成长 / 修复交易
- 对择时要求更高

示例：
- 科技成长类
- 创新药弹性方向

## 五、持仓角色定义
每个标的进入组合前，必须先定义角色：

### 1. 核心持仓候选
- 可承接较大权重
- 看重长期稳定性

### 2. 防守持仓候选
- 用于对冲风险偏好回落
- 强调股息 / 债性 / 现金流

### 3. 进攻持仓候选
- 用于提升收益弹性
- 看重成长 / 修复 / 风格切换

### 4. 观察名单
- 逻辑可跟踪，但暂不配置
- 等待变量验证

## 六、组合动作记录规则
每次动作至少记录以下 5 项：

1. **动作类型**
   - 加仓 / 减仓 / 建仓 / 清仓 / 继续观察

2. **动作对象**
   - 资产 / 行业 / 个股

3. **动作原因**
   - 由哪个宏观变量、知识判断或行业变化触发

4. **风险提示**
   - 最可能错在哪里

5. **后续验证点**
   - 未来看什么指标来验证这次动作是否正确

## 七、组合复盘机制
### 日度复盘
回答：
- 今天市场最重要变化是什么？
- 我的候选池里哪些标的发生了变化？
- 有没有新的风险信号？

### 周度复盘
对齐：
- `content/outputs/weekly-allocation-review-template.md`

重点复盘：
- 宏观判断是否正确
- 资产判断是否正确
- 行业选择是否正确
- 个股选择是否正确
- 仓位是否匹配环境

### 月度复盘
重点回答：
- 组合收益来自哪里
- 最大错误来自哪里
- 哪类资产 / 行业 / 个股贡献了 alpha
- 哪些动作属于无效忙碌
