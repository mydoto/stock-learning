# CONTENT-MAPPING.md

## 目的
本文件用于把 `stock-learning-site` 已有内容接入升级后的“全球资产配置研究系统”框架。

原则：
- 不推翻原内容
- 不重写已有知识卡片/行业/个股
- 先明确它们在新系统里的位置
- 后续再逐步补标签、补字段、补链接

---

## 一、现有内容在升级版中的定位

### 1. knowledge
目录：`content/knowledge/`

升级后定位：**底层原理库**

作用：
- 解释估值、盈利、现金流、周期、仓位、护城河等底层逻辑
- 为大类资产判断、行业研究、个股研究提供原理支持

建议新增标签：
- **影响资产**：权益 / 固收 / 商品 / 外汇 / 现金 / 另类
- **应用场景**：估值判断 / 周期判断 / 配置判断 / 风险控制

#### 已识别主题方向
- 利率与估值
- ROE / ROIC
- 自由现金流
- 商业模式 / 护城河
- 行业周期 / 景气度
- 仓位管理 / 回撤控制
- 预期差

### 2. sectors
目录：`content/sectors/`

升级后定位：**权益映射层**

作用：
- 把宏观变量和大类资产判断，映射到具体行业
- 解释行业顺风/逆风环境

建议新增字段：
- **宏观敏感变量**
- **适合的市场环境**
- **在组合中的角色**：进攻 / 防守 / 高股息 / 周期 / 成长

### 3. stocks
目录：`content/stocks/`

升级后定位：**标的观察层**

作用：
- 在行业主线和资产配置逻辑下，承接到具体公司

建议新增字段：
- **所属主线**
- **宏观敏感变量**
- **配置角色**：长期配置 / 观察名单 / 波段跟踪 / 防守标的

### 4. outputs
目录：`content/outputs/`

升级后定位：**输出复盘层**

作用：
- 承接每日观察、每周配置复盘、专题研究输出

建议分成三类：
- 每日观察
- 每周配置复盘
- 专题研究

---

## 二、升级后内容流

统一按这个路径组织：

1. **宏观变量**
2. **大类资产判断**
3. **行业映射**
4. **个股观察**
5. **组合配置**
6. **输出复盘**
7. **提醒执行**

现有内容的接入关系：
- `knowledge/` 支撑 1、2、5
- `sectors/` 支撑 3
- `stocks/` 支撑 4
- `outputs/` 支撑 6
- `REMINDER-*` 支撑 7
